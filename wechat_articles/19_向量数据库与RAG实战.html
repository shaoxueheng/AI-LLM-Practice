10.5 向量数据库：Milvus / Chroma / FAISS 对比实战

10.5.1 为什么需要向量数据库

当你有 10 万个文档块，每个块的向量维度是 1024 时，暴力搜索每次查询都要计算 10 万次余弦相似度——效率太低。向量数据库通过近似最近邻（ANN）算法加速搜索，将 O(n) 降低到 O(log n)。

"""使用 FAISS 构建向量索引"""

import faiss

import numpy as np

from sentence_transformers import SentenceTransformer

 

# 准备数据

embed_model = SentenceTransformer("BAAI/bge-large-zh-v1.5")

documents = [

    "退货政策：购买后7天内可无理由退货。",

    "退款流程：审核通过后3-5个工作日到账。",

    "配送说明：大部分商品3-5天送达。",

    "会员权益：金牌会员享受9折优惠。",

]

 

# 生成向量并建立索引

embeddings = embed_model.encode(documents)

dimension = embeddings.shape[1]  # 1024

 

# 创建 FAISS 索引（使用内积，等同于归一化后的余弦相似度）

faiss.normalize_L2(embeddings)

index = faiss.IndexFlatIP(dimension)  # 精确搜索

index.add(embeddings)

 

# 查询

query = "我想退货"

query_vec = embed_model.encode([query])

faiss.normalize_L2(query_vec)

 

distances, indices = index.search(query_vec, k=3)

print("检索结果:")

for dist, idx in zip(distances[0], indices[0]):

    print(f"  相似度: {dist:.3f} | {documents[idx]}")

10.6 检索策略优化：混合检索、重排序 Reranker

10.6.1 向量检索的局限

纯向量检索（稠密检索）并非万能。它在某些场景下表现不佳：

  • 精确关键词匹配：用户搜索"KB-2024-001"这样的编号
  • 专业术语：领域特有的缩写或术语
  • 数值查询：搜索"价格低于100元的商品"

10.6.3 RRF（Reciprocal Rank Fusion）融合算法

# 第10章/text

RRF 分数融合公式：

RRF_score(d) = sum( 1 / (k + rank_i(d)) )

 

其中：

- d 是候选文档

- rank_i(d) 是文档 d 在第 i 个检索器中的排名

- k 是平滑参数，通常取 60

 

示例：

文档A 在 BM25 中排第1，在向量检索中排第3：

RRF = 1/(60+1) + 1/(60+3) = 0.0164 + 0.0159 = 0.0323

 

文档B 在 BM25 中排第5，在向量检索中排第1：

RRF = 1/(60+5) + 1/(60+1) = 0.0154 + 0.0164 = 0.0318

 

最终排序：文档A > 文档B（两个检索器都认可的结果排名更高）

10.6.4 Reranker 重排序

重排序模型（Cross-Encoder）对初筛结果做精细打分。它同时输入查询和文档，输出相关性分数，精度远高于 Embedding 模型的独立编码。

"""使用 Reranker 进行重排序"""

from sentence_transformers import CrossEncoder

 

# 加载重排序模型

reranker = CrossEncoder("BAAI/bge-reranker-v2-m3", max_length=512)

 

query = "退货需要什么条件？"

# 初检索结果（模拟）

candidates = [

    "退货政策：购买后7天内可无理由退货，需保持商品完好。",

    "配送说明：大部分商品3-5天送达，偏远地区可能延迟。",

    "退款流程：提交退款申请后，审核通过3-5个工作日到账。",

    "退换货注意事项：食品类、贴身衣物不支持无理由退换。",

    "会员权益：金牌会员享受全场9折优惠。",

]

 

# 重排序

pairs = [(query, doc) for doc in candidates]

scores = reranker.predict(pairs)

 

# 按分数降序排列

ranked = sorted(zip(candidates, scores), key=lambda x: x[1], reverse=True)

 

print(f"查询: {query}\n")

print("重排序后结果:")

for doc, score in ranked:

    print(f"  分数: {score:.4f} | {doc}")

10.7.2 评估指标详解

检索层评估：

# 第10章/text

Recall@K（召回率）= 被检索到的相关文档数 / 相关文档总数

 

示例：

- 知识库中有3篇关于退货的文档

- Top-5 检索结果中包含了其中2篇

- Recall@5 = 2/3 = 0.667

 

Precision@K（精确率）= 检索结果中相关文档数 / K

 

示例：

- Top-5 检索结果中有2篇是关于退货的

- Precision@5 = 2/5 = 0.4

 

MRR（平均倒数排名）= 1 / 第一个相关文档的排名

 

示例：

- 第一个相关文档排在第2位

- MRR = 1/2 = 0.5

# 第10章/python

"""

企业知识库问答系统

基于 RAG 架构，支持多格式文档导入和智能问答

"""

 

import os

import json

from pathlib import Path

from typing import List, Dict, Optional

from dataclasses import dataclass

 

import chromadb

from sentence_transformers import SentenceTransformer, CrossEncoder

from openai import OpenAI

 

 

@dataclass

class RetrievedChunk:

    """检索到的文档块"""

    content: str

    source: str

    score: float

 

 

class KnowledgeBaseQA:

    """企业知识库问答系统"""

 

    def __init__(

        self,

        embedding_model: str = "BAAI/bge-large-zh-v1.5",

        reranker_model: str = "BAAI/bge-reranker-v2-m3",

        llm_model: str = "qwen-plus",

        db_path: str = "./knowledge_db",

    ):

        print("正在初始化系统...")

 

        # Embedding 模型

        self.embedder = SentenceTransformer(embedding_model)

 

        # Reranker 模型

        self.reranker = CrossEncoder(reranker_model, max_length=512)

 

        # 向量数据库

        self.chroma_client = chromadb.PersistentClient(path=db_path)

        self.collection = self.chroma_client.get_or_create_collection(

            name="enterprise_kb",

            metadata={"hnsw:space": "cosine"},

        )

 

        # LLM 客户端

        self.llm_client = OpenAI(

            api_key=os.getenv("DASHSCOPE_API_KEY"),

            base_url="https://dashscope.aliyuncs.com/compatible-mode/v1",

        )

        self.llm_model = llm_model

 

        # 文本切分器

        self.splitter = RecursiveTextSplitter(chunk_size=500, chunk_overlap=100)

 

        print("系统初始化完成！")

 

    # ===== 文档导入 =====

    def ingest_documents(self, doc_dir: str):

        """批量导入目录下的所有文档"""

        doc_path = Path(doc_dir)

        parser = DocumentParser()

 

        all_chunks = []

        for file in doc_path.rglob("*"):

            if file.suffix.lower() in [".pdf", ".docx", ".md", ".txt", ".html"]:

                print(f"正在解析: {file.name}")

                try:

                    text = parser.parse(str(file))

                    chunks = self.splitter.split(text)

                    for i, chunk in enumerate(chunks):

                        all_chunks.append({

                            "id": f"{file.stem}_{i}",

                            "content": chunk,

                            "source": file.name,

                        })

                except Exception as e:

                    print(f"  解析失败: {e}")

 

        if not all_chunks:

            print("没有找到可解析的文档。")

            return

 

        # 生成向量并存入数据库

        print(f"共 {len(all_chunks)} 个文档块，正在生成向量...")

        contents = [c["content"] for c in all_chunks]

        embeddings = self.embedder.encode(contents, show_progress_bar=True).tolist()

 

        self.collection.add(

            ids=[c["id"] for c in all_chunks],

            documents=contents,

            embeddings=embeddings,

            metadatas=[{"source": c["source"]} for c in all_chunks],

        )

        print(f"导入完成！共 {len(all_chunks)} 个文档块已索引。")

 

    # ===== 检索 =====

    def retrieve(self, query: str, top_k: int = 10, rerank_top_n: int = 5) -> List[RetrievedChunk]:

        """检索 + 重排序"""

        # 1. 向量检索

        query_embedding = self.embedder.encode([query]).tolist()

        results = self.collection.query(

            query_embeddings=query_embedding,

            n_results=top_k,

        )

 

        if not results["documents"][0]:

            return []

 

        # 2. 重排序

        candidates = results["documents"][0]

        sources = [m["source"] for m in results["metadatas"][0]]

 

        pairs = [(query, doc) for doc in candidates]

        rerank_scores = self.reranker.predict(pairs)

 

        # 3. 按分数排序

        ranked = sorted(

            zip(candidates, sources, rerank_scores),

            key=lambda x: x[2],

            reverse=True,

        )

 

        return [

            RetrievedChunk(content=doc, source=src, score=float(score))

            for doc, src, score in ranked[:rerank_top_n]

        ]

 

    # ===== 生成回答 =====

    def answer(self, question: str) -> str:

        """RAG 问答：检索 + 生成"""

        # 1. 检索相关文档

        chunks = self.retrieve(question)

 

        if not chunks:

            return "抱歉，知识库中没有找到相关信息。"

 

        # 2. 构造上下文

        context = "\n\n".join(

            f"【来源: {c.source}】{c.content}" for c in chunks

        )

 

        # 3. 构造 Prompt

        system_prompt = """你是企业知识库助手。请根据以下参考文档回答用户问题。

规则：

1. 只使用参考文档中的信息来回答

2. 如果文档中没有相关信息，请明确告知用户

3. 回答要简洁、准确、有条理

4. 在回答末尾注明信息来源"""

 

        user_prompt = f"""参考文档：

{context}

 

用户问题：{question}

 

请根据参考文档回答："""

 

        # 4. 调用 LLM 生成

        response = self.llm_client.chat.completions.create(

            model=self.llm_model,

            messages=[

                {"role": "system", "content": system_prompt},

                {"role": "user", "content": user_prompt},

            ],

            temperature=0.3,

            max_tokens=1024,

        )

 

        return response.choices[0].message.content

 

 

# ===== 使用示例 =====

if __name__ == "__main__":

    qa = KnowledgeBaseQA()

 

    # 导入文档

    qa.ingest_documents("./docs")

 

    # 问答测试

    questions = [

        "退货政策是什么？",

        "金牌会员有什么权益？",

        "如何申请发票？",

    ]

 

    for q in questions:

        print(f"\n问: {q}")

        answer = qa.answer(q)

        print(f"答: {answer}")

        print("-" * 50)

关键收获：

1. RAG 通过"检索 + 生成"解决了大模型的知识时效性和幻觉问题
