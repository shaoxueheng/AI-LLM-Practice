第11章 LangChain 应用开发框架

核心比喻：如果大模型是一颗强劲的发动机，那 LangChain 就是围绕它搭建的汽车底盘——它提供方向盘（Prompt 管理）、变速箱（Chain 链式调用）、仪表盘（调试追踪）和导航系统（Agent 决策），让你不必从零造车，就能快速上路。

11.1 LangChain 核心概念与架构总览

11.1.1 LangChain 是什么

LangChain 是一个基于 LLM 的应用开发框架，由 Harrison Chase 于 2022 年底创建。它解决了一个核心问题：如何将大模型与外部工具、数据源、业务逻辑串联起来，构建完整的 AI 应用。

11.2 Model I/O：模型调用的统一抽象

11.2.1 适配器模式

LangChain 的 Model I/O 模块采用了经典的适配器模式：无论底层是哪个模型供应商，对外暴露统一的接口。

图11-4

11.2.2 基础用法

# 第11章/python
"""LangChain Model I/O 基础用法"""
from langchain_openai import ChatOpenAI
from langchain_core.messages import HumanMessage, SystemMessage
 
# 初始化模型（统一接口）
llm = ChatOpenAI(
    model="gpt-4o-mini",
    temperature=0.7,
    max_tokens=1024,
)
 
# 调用模型
messages = [
    SystemMessage(content="你是一位友善的AI助手。"),
    HumanMessage(content="用一句话解释什么是机器学习。"),
]
 
response = llm.invoke(messages)
print(response.content)
 
# 切换到 Anthropic Claude（只需改一行）
from langchain_anthropic import ChatAnthropic
 
llm_claude = ChatAnthropic(model="claude-sonnet-4-20250514", temperature=0.7)
response = llm_claude.invoke(messages)
print(response.content)

11.2.3 Prompt Templates

Prompt 模板让你将模板逻辑与变量数据分离：

# 第11章/python
"""Prompt Templates 使用示例"""
from langchain_core.prompts import ChatPromptTemplate
 
# 定义模板
prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一位{domain}领域的专家。请用{style}的风格回答问题。"),
    ("human", "{question}"),
])
 
# 填充变量
formatted = prompt.invoke({
    "domain": "金融",
    "style": "简洁专业",
    "question": "什么是市盈率？",
})
 
print(formatted)
# 输出：
# [SystemMessage(content='你是一位金融领域的专家。请用简洁专业的风格回答问题。'),
#  HumanMessage(content='什么是市盈率？')]

11.2.4 Output Parsers

让 LLM 的输出自动解析为结构化数据：

# 第11章/python
"""Output Parsers 使用示例"""
from langchain_core.output_parsers import JsonOutputParser
from langchain_core.prompts import ChatPromptTemplate
from langchain_openai import ChatOpenAI
from pydantic import BaseModel, Field
 
# 定义输出结构
class BookReview(BaseModel):
    title: str = Field(description="书名")
    rating: int = Field(description="评分1-5")
    summary: str = Field(description="一句话简评")
    tags: list[str] = Field(description="标签列表")
 
# 创建解析器
parser = JsonOutputParser(pydantic_object=BookReview)
 
# 构建 Prompt（包含格式说明）
prompt = ChatPromptTemplate.from_messages([
    ("system", "你是书评助手。{format_instructions}"),
    ("human", "请评价《{book}》"),
])
 
# 组装链
llm = ChatOpenAI(model="gpt-4o-mini")
chain = prompt | llm | parser
 
# 调用
result = chain.invoke({
    "book": "三体",
    "format_instructions": parser.get_format_instructions(),
})
 
print(result)
# {'title': '三体', 'rating': 5, 'summary': '...', 'tags': ['科幻', '硬科幻', '宇宙']}

11.3 Chain 链式调用与 LCEL 表达式语言

11.3.1 什么是 LCEL

LCEL（LangChain Expression Language）是 LangChain 的核心编程范式，它使用 | 管道操作符将各个组件串联成一条处理链——就像 Linux 中的管道命令。

11.3.3 基础 LCEL 链

# 第11章/python
"""LCEL 基础示例"""
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser
 
# 组件定义
prompt = ChatPromptTemplate.from_messages([
    ("system", "你是翻译助手，将中文翻译为{target_lang}。"),
    ("human", "{text}"),
])
llm = ChatOpenAI(model="gpt-4o-mini")
parser = StrOutputParser()
 
# 用 | 管道操作符组装链
chain = prompt | llm | parser
 
# 同步调用
result = chain.invoke({"target_lang": "英文", "text": "今天天气真好"})
print(result)  # "The weather is really nice today."
 
# 流式输出
for chunk in chain.stream({"target_lang": "日语", "text": "你好世界"}):
    print(chunk, end="", flush=True)
 
# 批量处理
results = chain.batch([
    {"target_lang": "英文", "text": "苹果"},
    {"target_lang": "英文", "text": "香蕉"},
    {"target_lang": "英文", "text": "橘子"},
])

11.3.4 复杂链：并行与分支

图11-7

# 第11章/python
"""LCEL 并行链示例"""
from langchain_core.runnables import RunnableParallel, RunnablePassthrough
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser
 
llm = ChatOpenAI(model="gpt-4o-mini")
parser = StrOutputParser()
 
# 定义三条并行链
summary_chain = (
    ChatPromptTemplate.from_template("请用一句话概括以下文本:\n{text}")
    | llm | parser
)
 
keywords_chain = (
    ChatPromptTemplate.from_template("提取以下文本的5个关键词（逗号分隔）:\n{text}")
    | llm | parser
)
 
sentiment_chain = (
    ChatPromptTemplate.from_template("分析以下文本的情感倾向（积极/消极/中性）:\n{text}")
    | llm | parser
)
 
# 并行执行
parallel_chain = RunnableParallel(
    summary=summary_chain,
    keywords=keywords_chain,
    sentiment=sentiment_chain,
)
 
# 最终汇总链
report_prompt = ChatPromptTemplate.from_template("""
基于以下分析结果，生成一份结构化报告：
- 摘要: {summary}
- 关键词: {keywords}
- 情感: {sentiment}
""")
 
full_chain = (
    {"text": RunnablePassthrough()}
    | parallel_chain
    | report_prompt
    | llm
    | parser
)
 
result = full_chain.invoke("今天公司发布了新产品，市场反响非常热烈...")
print(result)

11.3.5 LCEL 的核心优势

表11-1　

特性说明
**流式输出**所有链自动支持 `.stream()`
**异步支持**所有链自动支持 `.ainvoke()`
**批量处理**所有链自动支持 `.batch()`

# 第11章/python
"""LCEL 容错与重试"""
from langchain_openai import ChatOpenAI
from langchain_anthropic import ChatAnthropic
 
# 主模型 + 备用模型
primary = ChatOpenAI(model="gpt-4o").with_retry(stop_after_attempt=3)
fallback = ChatAnthropic(model="claude-sonnet-4-20250514")
 
# 主模型失败时自动切换到备用模型
robust_llm = primary.with_fallbacks([fallback])

11.4 Memory 记忆模块：对话历史管理

11.4.1 为什么需要 Memory

LLM 本身是无状态的——每次调用都是全新的。要实现多轮对话，必须手动管理历史消息。LangChain 的 Memory 模块封装了这一过程。

11.4.4 代码示例：构建有记忆的聊天机器人

# 第11章/python
"""带记忆的聊天机器人"""
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder
from langchain_core.runnables.history import RunnableWithMessageHistory
from langchain_community.chat_message_histories import ChatMessageHistory
 
# 存储每个会话的历史
session_store = {}
 
def get_session_history(session_id: str):
    if session_id not in session_store:
        session_store[session_id] = ChatMessageHistory()
    return session_store[session_id]
 
# 构建链
prompt = ChatPromptTemplate.from_messages([
    ("system", "你是一个友善的AI助手。"),
    MessagesPlaceholder(variable_name="history"),
    ("human", "{input}"),
])
 
llm = ChatOpenAI(model="gpt-4o-mini")
chain = prompt | llm
 
# 包装为带记忆的链
chain_with_memory = RunnableWithMessageHistory(
    chain,
    get_session_history,
    input_messages_key="input",
    history_messages_key="history",
)
 
# 多轮对话
config = {"configurable": {"session_id": "user_001"}}
 
r1 = chain_with_memory.invoke({"input": "我叫小明"}, config=config)
print(f"AI: {r1.content}")
 
r2 = chain_with_memory.invoke({"input": "我刚才说我叫什么？"}, config=config)
print(f"AI: {r2.content}")  # "你刚才说你叫小明。"
