15.1 模型量化技术：INT8、INT4、GPTQ、AWQ

15.1.1 为什么需要量化

一个70亿参数的模型（7B），使用FP16存储需要约14GB显存。如果是70B模型，则需要140GB -- 这已经超出单张消费级GPU的容量。量化技术通过降低数值精度来"压缩"模型。

图15-1

15.1.2 量化的直觉理解

量化的本质可以类比为地图的精度缩放：

一张1:1000的高精度地图（FP16），包含每棵树、每块砖的位置。将它"量化"为1:50000的导航地图（INT4），虽然丢失了一些细节，但主要道路和方向完全保留，而且地图变小了很多倍，翻阅（推理）速度也快了很多。

15.1.4 使用 AutoGPTQ 量化模型

# 第15章/python
from transformers import AutoTokenizer
from auto_gptq import AutoGPTQForCausalLM, BaseQuantizeConfig
import torch
 
# 1. 加载原始模型和分词器
model_name = "Qwen/Qwen2.5-7B-Instruct"
tokenizer = AutoTokenizer.from_pretrained(model_name, trust_remote_code=True)
 
# 2. 配置量化参数
quantize_config = BaseQuantizeConfig(
    bits=4,              # 量化位数
    group_size=128,      # 分组大小（每128个参数共享缩放因子）
    desc_act=True,       # 激活感知排序（提升精度）
    damp_percent=0.1,    # 阻尼系数
)
 
# 3. 加载模型
model = AutoGPTQForCausalLM.from_pretrained(
    model_name,
    quantize_config=quantize_config,
    torch_dtype=torch.float16,
    device_map="auto"
)
 
# 4. 准备校准数据（用于量化过程中计算最优缩放因子）
calibration_texts = [
    "人工智能正在改变我们的生活方式。",
    "深度学习是机器学习的一个重要分支。",
    "大语言模型具有强大的文本生成能力。",
    # ... 通常需要128-256条样本
]
 
calibration_data = []
for text in calibration_texts:
    input_ids = tokenizer(text, return_tensors="pt").input_ids
    calibration_data.append({"input_ids": input_ids})
 
# 5. 执行量化
model.quantize(calibration_data)
 
# 6. 保存量化模型
model.save_quantized("./qwen2.5-7b-gptq-int4")
tokenizer.save_pretrained("./qwen2.5-7b-gptq-int4")
 
print("量化完成! 模型大小从约14GB降至约4GB")

15.3 KV Cache 优化与长上下文推理

15.3.1 KV Cache 是什么

在Transformer的自回归生成过程中，每生成一个新Token，都需要计算与所有历史Token的注意力。KV Cache将已计算的Key和Value缓存起来，避免重复计算。

15.4.4 vLLM 多GPU推理配置

# 第15章/python
from vllm import LLM, SamplingParams
 
# 张量并行：将模型拆分到4张GPU
llm = LLM(
    model="Qwen/Qwen2.5-72B-Instruct",
    tensor_parallel_size=4,      # 4卡张量并行
    dtype="auto",                # 自动选择精度
    gpu_memory_utilization=0.9,  # GPU显存利用率
    max_model_len=32768,         # 最大序列长度
    trust_remote_code=True,
)
 
# 推理
prompts = [
    "解释量子纠缠的原理",
    "写一首关于AI的诗",
    "分析全球经济趋势",
]
 
sampling_params = SamplingParams(
    temperature=0.7,
    top_p=0.9,
    max_tokens=1024,
)
 
outputs = llm.generate(prompts, sampling_params)
 
for output in outputs:
    print(f"Prompt: {output.prompt[:30]}...")
    print(f"Output: {output.outputs[0].text[:100]}...")
    print("---")
