9.1 为什么需要微调：通用能力 vs 专业能力

9.1.1 通才与专才的困境

想象你经营一家法律咨询公司。你聘请了一位"百事通"——他博览群书、无所不知，但当客户问"根据《民法典》第1176条，自甘风险的免责条件有哪些？"时，他只能给出模糊的回答。

大模型面临同样的困境。ChatGPT 和 Qwen 这样的通用模型，虽然在预训练阶段学习了海量知识，但在特定领域往往表现不够精准。

9.1.4 微调的典型场景

1. 风格统一：让模型始终以某种口吻回答（如萌系客服、严肃法律顾问）

2. 格式规范：输出始终符合特定 JSON Schema 或表格格式

3. 领域增强：医疗问诊、法律咨询、金融分析等垂直领域

4. 安全合规：让模型遵循企业特定的内容安全策略

5. 语言适配：增强模型对小语种或方言的理解能力

9.2 全参数微调 vs 参数高效微调

9.2.1 冰山模型：理解参数规模

一个 7B（70亿参数）的大模型，如果使用 float32 精度存储，光模型权重就需要 28GB 显存。加上优化器状态、梯度等，全参数微调需要的显存是模型本身的 4-6 倍。

图9-4

关键洞察：对于 7B 模型的全参数微调，你大约需要 4 张 A100 80GB 显卡——这对个人开发者或中小企业来说成本过高。

9.2.2 参数高效微调（PEFT）的核心思想

PEFT 的核心理念是：不动大模型的绝大部分参数，只训练一小部分新增参数。这就像你不需要重建一栋大楼来改造一个房间——只需局部装修。

9.3 LoRA 与 QLoRA：低成本微调技术详解

9.3.1 LoRA 的数学直觉

LoRA（Low-Rank Adaptation）的核心假设是：微调过程中权重的变化量是低秩的。换句话说，虽然模型有数十亿参数，但微调只需要在一个低维子空间中调整。

这就像调收音机的频道——收音机内部有数百个电子元件，但你只需要转一个旋钮（低维操作）就能切换到目标频道。

数学表达：

原始权重矩阵 W 的更新过程：

# 第9章/text
W' = W + delta_W = W + B × A
 
其中：
- W 是原始权重矩阵，维度为 d × k（如 4096 × 4096）
- delta_W 是更新量
- B 的维度为 d × r（如 4096 × 8）
- A 的维度为 r × k（如 8 × 4096）
- r 是秩（rank），通常取 4、8、16、32、64

图9-7

参数效率计算：

  • 原始矩阵参数量：4096 x 4096 = 16,777,216
  • LoRA 新增参数量：4096 x 8 + 8 x 4096 = 65,536
  • 压缩比：65,536 / 16,777,216 = 0.39%

9.3.3 QLoRA：量化 + LoRA 的组合拳

QLoRA 在 LoRA 基础上引入了三项关键创新：

1. 4-bit NormalFloat (NF4) 量化：将模型权重从 16-bit 压缩到 4-bit

2. 双重量化（Double Quantization）：对量化常数本身再做一次量化

3. 分页优化器（Paged Optimizer）：利用 CPU 内存处理 GPU 显存溢出

9.4.1 数据决定微调效果的上限

9.4.5 使用 GPT 辅助生成训练数据

当人工编写数据量不足时，可以用强模型辅助生成，但需注意：

# 第9章/python
# 使用强模型生成训练数据的示例
import json
from openai import OpenAI
 
client = OpenAI()
 
def generate_training_pair(domain: str, seed_example: str) -> dict:
    """利用 GPT-4 生成高质量训练数据对"""
    prompt = f"""你是一名数据标注专家。请根据以下示例，生成一条新的【{domain}】领域的训练数据。
 
示例：{seed_example}
 
要求：
1. 问题要自然、多样，模拟真实用户提问
2. 答案要准确、专业、简洁
3. 不要照搬示例，要有创造性
4. 输出 JSON 格式：{{"instruction": "...", "output": "..."}}
"""
    response = client.chat.completions.create(
        model="gpt-4o",
        messages=[{"role": "user", "content": prompt}],
        temperature=0.8,  # 适当提高多样性
    )
    return json.loads(response.choices[0].message.content)
警告：GPT 生成的数据必须经过人工审核。直接使用未审核的 AI 生成数据会导致"模型退化"（Model Collapse）现象——让小模型学大模型的错误。
